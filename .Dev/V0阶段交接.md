# SillyTender V0 阶段交接

## 阶段结果

V0 已建立 SillyTender 的 SillyTavern 第三方扩展骨架：根级 manifest、浏览器 ES module 入口、基础设置持久化、顶部菜单栏独立抽屉、扩展设置面板、路线图占位卡片、版本检查提示与响应式样式。

## 已完成内容

- `manifest.json` 声明扩展为 `SillyTender`，版本保持 `0.0.1`。
- `index.js` 初始化设置、UI 与 `window.SillyTender` 调试入口。
- `core/settings.js` 合并默认设置并保存到 `extension_settings['st-SillyTender']`。
- `ui/drawer.js` 加载模板、插入顶部抽屉、绑定页签与打开按钮。
- `ui/templates/drawer.html` 提供“概览 / 功能规划 / 设置”三页。
- `ui/templates/settings.html` 提供扩展设置面板入口。
- `ui/placeholders.js` 展示 V1-V5 功能规划占位。
- `ui/version-check.js` 检查远端 manifest 版本，并在失败时降级为温和提示。
- `style.css` 提供主题变量样式、紧凑密度与手机布局适配。

## 已知限制

- V0 不实现提示词调度、破甲词、长期记忆、外部知识库或角色卡广场。
- V0 不注册聊天事件、宏、模型请求中间件或外部服务连接。
- 当前仓库没有 npm 脚本、lint、test runner；自动验证以静态检查为主。
- 需要在 SillyTavern 浏览器环境中手动确认实际 drawer 动画和持久化表现。

## 建议验证

1. 使用用户已建立的 Docker SillyTavern 扩展符号链接验证：
   `H:\DockerContainer\SillyTavern\extensions\st-SillyTender`
2. 启动或刷新 Docker 部署的 SillyTavern。
3. 刷新浏览器并确认：
   - Extensions 中显示 SillyTender。
   - 顶部栏出现 SillyTender 图标。
   - 抽屉可打开/关闭。
   - 三个页签可切换。
   - 设置保存后刷新仍保持。
   - 控制台没有阻塞加载的错误。
4. 如要确认最低兼容，在 `SillyTavern-1.15.0` 重复最小加载验证。
