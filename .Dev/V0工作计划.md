# SillyTender V0 工作计划

## 范围

V0 只建立 `st-SillyTender` 的 SillyTavern 第三方扩展骨架与 UI 框架，不实现后续业务系统。

## 目标

- 提供可被 SillyTavern 加载的 `manifest.json`、`index.js`、`style.css`。
- 在顶部菜单栏加入 SillyTender 独立抽屉入口。
- 在抽屉中提供“概览 / 功能规划 / 设置”三页。
- 在扩展设置面板提供基础设置块与“打开 SillyTender”按钮。
- 使用 `extension_settings['st-SillyTender']` 持久化 V0 基础配置。
- 保留 V1-V5 功能规划占位，但不接入提示词、数据库、外部服务或角色卡业务。

## 文件结构

- `manifest.json`：SillyTavern 扩展声明，版本固定为 `0.0.1`。
- `index.js`：扩展入口与 bootstrap。
- `style.css`：V0 UI 样式与移动端适配。
- `core/constants.js`：扩展名称、版本、路径、仓库地址等常量。
- `core/settings.js`：默认设置、设置合并、控件绑定与保存。
- `ui/drawer.js`：模板加载、顶部抽屉插入、页签切换与事件绑定。
- `ui/placeholders.js`：V1-V5 路线图占位卡片。
- `ui/version-check.js`：远端 manifest 版本检查与 UI 提示。
- `ui/templates/settings.html`：扩展设置面板模板。
- `ui/templates/drawer.html`：顶部独立抽屉模板。

## 非目标

- 不修改 SillyTavern 提示词插入链路。
- 不注册宏、消息事件或模型请求中间件。
- 不创建后端服务、数据库、向量存储或外部知识库连接。
- 不实现破甲词、长期记忆、RAG、角色卡广场等业务功能。
- 不自动修改 `0.0.1` 版本号。

## 兼容性策略

- 仅使用 SillyTavern 1.15.0 已存在的扩展 API：`extension_settings`、`renderExtensionTemplateAsync()` 与 `saveSettingsDebounced()`。
- 模板加载优先使用 `renderExtensionTemplateAsync('third-party/st-SillyTender/ui/templates', id)`，失败时 fallback 到 `$.get()`。
- 抽屉交互使用 SillyTavern 既有 `drawer` / `drawer-toggle` / `drawer-content` / `closedIcon` / `openIcon` / `closedDrawer` / `openDrawer` 类约定。
- CSS 使用 SillyTavern 主题变量与响应式布局，兼容 PC 横版与手机竖版。

## 验证清单

1. JSON 解析 `manifest.json` 成功，版本为 `0.0.1`。
2. JavaScript 文件无明显语法错误，import 路径符合第三方扩展目录层级。
3. 通过用户已建立的符号链接在 Docker 部署中验证：`H:\DockerContainer\SillyTavern\extensions\st-SillyTender`。
4. 顶部栏出现 SillyTender 图标，点击可打开/关闭抽屉。
5. “概览 / 功能规划 / 设置”页签可切换。
6. 设置开关与 UI 密度刷新后可保持。
7. 更新检查失败时不阻塞扩展加载。
8. 如需要最低兼容确认，在 `SillyTavern-1.15.0` 重复最小加载验证。

## 后续阶段

V1 应从 `提示词调度（Inner Universe）` 开始，需要在进入实现前重新审查 `.Dev/sillytender-prompt-middleware-research.md` 与 prompt 架构标准，明确 SillyTavern 事件挂载点、提示词注入边界、缓存命中策略、设置隔离和回滚方案。
